在Python中定义一个函数def fun()后，直接调用fun返回的是函数的内存地址，调用fun()返回的是函数的return值。

荧光定量（qPCR）原理、实验流程及结果分析

基本概念

PCR定义

PCR：聚合酶链式反应（polymerase chain reaction, PCR）是利用一段DNA为模板，在DNA聚合酶和核苷酸底物共同参与下，将该段DNA扩增至足够数量，以便进行结构和功能分析。

QPCR定义

Real-time PCR，是指在PCR体系中加入荧光染料或荧光探针，利用荧光信号积累实时监测整个PCR进程，最后通过CT值和标准曲线对未知模板进行定量分析的方法。

QPCR化学原理

荧光（染料、荧光基团）

非特异性荧光标记：SYBR Green

  1. 一种DNA小沟非饱和性结合染料
  2. 与DNA结合时发光/不结合（游离）时不发光
  3. 每形成一条DNA双链，就会有一定数量的染料结合上去
  4. 染料一结合，就会产生荧光信号
  5. 信号强度与DNA分子总数目成正比

特异性荧光标记：TaqMan Probe

  1. 一种水解探针（5’ Reporter, 3’ Quencher）
  2. 完整时不发光/水解后发光
  3. 每形成一条DNA链，就会水解一条探针
  4. 每水解一条探针，就会产生一个单位信号
  5. 信号强度与结合过探针的DNA分子总数目成正比

SYBR Green VS TaqMan Probe

优点 缺点
SYBR Green 价格便宜、适用性广、操作方便 特异性高、支持多重荧光定量
TaqMan Probe 引物物要求高、特异性稍差、不能进行多重定量 价格贵、只适合特定目标

PCA主成分析原理、理解和代码实现

背景

在用统计分析方法研究多变量的课题时，变量个数太多就会增加课题的复杂性。人们自然希望变量个数较少而得到的信息较多。

在很多情形，变量之间是有一定的相关关系的，当两个变量之间有一定相关关系时，可以解释为这两个变量反映此课题的信息有一定的重叠。

在许多领域的研究与应用中，通常需要对含有多个变量的数据进行观测，收集大量数据后进行分析寻找规律。多变量大数据集无疑会为研究和应用提供丰富的信息，但是也在一定程度上增加了数据采集的工作量。更重要的是在很多情形下，许多变量之间可能存在相关性，从而增加了问题分析的复杂性。如果分别对每个指标进行分析，分析往往是孤立的，不能完全利用数据中的信息，因此盲目减少指标会损失很多有用的信息，从而产生错误的结论。

因此需要找到一种合理的方法，在减少需要分析的指标同时，尽量减少原指标包含信息的损失，以达到对所收集数据进行全面分析的目的。由于各变量之间存在一定的相关关系，因此可以考虑将关系紧密的变量变成尽可能少的新变量，使这些新变量是两两不相关的，那么就可以用较少的综合指标分别代表存在于各个变量中的各类信息。主成分分析与因子分析就属于这类降维算法。

概念

PCA（Principal Component Analysis），即主成分分析方法，是一种使用最广泛的数据降维算法。PCA的主要思想是将n维特征映射到k维上，这k维是全新的正交特征也被称为主成分，是在原有n维特征的基础上重新构造出来的k维特征。PCA的工作就是从原始的空间中顺序地找一组相互正交的坐标轴，新的坐标轴的选择与数据本身是密切相关的。其中，第一个新坐标轴选择是原始数据中方差最大的方向，第二个新坐标轴选取是与第一个坐标轴正交的平面中使得方差最大的，第三个轴是与第1,2个轴正交的平面中方差最大的。依次类推，可以得到n个这样的坐标轴。通过这种方式获得的新的坐标轴，我们发现，大部分方差都包含在前面k个坐标轴中，后面的坐标轴所含的方差几乎为0。于是，我们可以忽略余下的坐标轴，只保留前面k个含有绝大部分方差的坐标轴。事实上，这相当于只保留包含绝大部分方差的维度特征，而忽略包含方差几乎为0的特征维度，实现对数据特征的降维处理。

讲人话

主成分分析法是运用“降维”思想，把多个指标变换成少数综合指标的多元统计方法，这里的综合指标就是主成分。每个主成分都是原始变量的线性组合，彼此相互独立，并保留了原始变量绝大部分信息。其本质是通过原始变量的相关性，寻求相关变量的综合替代对象，并且保证了转化过程中的信息损失最小。

背景和概念看不懂？没关系！我来用人话讲解一遍。

PCA主成分析可以用来解决什么问题：

  1. 存在很多个个体、很多个变量，你想通过变量来对个体进行区分，找出个体之间的差别；
  2. 变量太多太多了，不可能为了区分每两个个体之间都需要用上所有的变量，变量与变量之间反应的信息也许还会有所重叠。

PCA主成分析的目标：只保留一个轴的时候（二维降到一维），信息保留最多。

怎么样最好：找到数据分布最分散的方向（方差最大）作为主成分（坐标轴）

首先：去中心化（把坐标原点放在数据中心），也就是数据的中心化与标准化，可以看这篇文章：数据中心化与标准化。

然后：找坐标系（找到方差最大的方向），问题就是怎么找到方差最大的方向？（这一步是通过算法实现的，没看懂，需要数学线性代数的知识，我的都还给老师了，不过对于非计算机专业的，只是想使用PCA画图，这一步完全可以跳过）

找到一个能够反应尽量多的点信息的坐标，同时也要使得在新坐标下，点与点之间的差异性可以得到最大的保留。

主成分与原始变量之间的关系：

  • 主成分是原始变量的线性组合；
  • 主成分的数量相对于原始数量更少；
  • 主成分保留了原始变量的大部分信息。

PCoA

PCoA（Principal Co-ordinates Analysis）分析即主坐标分析，可呈现研究数据相似性或差异性的可视化坐标，是一种非约束性的数据降维分析方法，可用来研究样本群落组成的相似性或相异性。它与PCA类似，通过一系列的特征值和特征向量进行排序后，选择主要排在前几位的特征值，找到距离矩阵中最主要的坐标，结果是数据矩阵的一个旋转，它没有改变样本点之间的相互位置关系，只是改变了坐标系统。

两者之间的区别：PCA是基于样本的相似系数矩阵(如欧式距离)来寻找主成分，而PCoA是基于距离矩阵(欧式距离以外的其他距离)来寻找主坐标。

5、通过分析成分矩阵，得出主成分成分公式与权重；

6、输出主成分分析法综合得分。

注意事项

  1. 主成分要求变量之间的共线性或相关关系比较强，否则不能通过 KMO 检验和 Bartlett 球形检验；
  2. 主成分分析倾向于降维，从而达到简化系统结构，抓住问题实质的目的。（可侧重于输出结果 2、输出结果 3、输出结果 8）；
  3. 主成分分析时通常需要综合自己的专业知识，以及软件结果进行综合判断，即使是特征根值小于 1，也一样可以提取主成分；
  4. KMO 值为 null 不存在可能导致的原因为：
    • 样本量过少容易导致相关系数过高，一般希望分析样本量大于 5 倍分析项个数；
    • 各个分析项之间的相关关系过高或过低。

数据框是一种表格式的数据结构。数据框旨在模拟数据集，与其他统计软件例如SAS或者SPSS中的数据集的概念一致。

缺失数据

缺失数据分类

统计学家通常将缺失数据分为三类。它们都用概率术语进行描述，但思想都非常直观。我们将用sleep研究中对做梦时长的测量（有12个动物有缺失值）来依次阐述三种类型。

完全随机缺失：若某变量的缺失数据与其他任何观测或未观测变量都不相关，则数据为完全随机缺失（MCAR）。若12个动物的做梦时长值缺失不是由于系统原因，那么可认为数据是（MCAR）。注意，如果每个有缺失值的变量都是MCAR，那么可以将数据完整的实例看做是对更大数据集的一个简单随机抽样。

随机缺失：若某变量上的缺失数据与其他观测变量相关，与它自己的未观测值不相关，则数据为随机缺失(MAR)。例如，体重较小的动物更可能有做梦时长的缺失值（可能因为较小的动物较难观察）“缺失”与动物的做梦时长无关，那么该数据就可以认为是MAR。此时，一旦你控制了体重变量，做梦时长数据的缺失与出现将是随机的。

非随机缺失：若缺失数据不属于MCAR或MAR，则数据为非随机缺失（NMAR）。例如，做梦时长越短的动物也更可能有做梦数据的缺失（可能由于难以测量时长较短的事件），那么数据可认为是NMAR。

为何会出现缺失数据？

  1. 机器断电，设备故障导致某个测量值发生了丢失。
  2. 测量根本没有发生，例如在做调查问卷时，有些问题没有回答，或者有些问题是无效的回答等。

在R中，NA代表缺失值，NA是不可用，not available的简称，用来存储缺失信息。

这里缺失值NA表示没有，但注意没有并不一定就是0，NA是不知道是多少，也能是0，也可能是任何值，缺失值和值为零是完全不同的。

人生第一台DIY主机装配单分享

缘由

Book3的电池过完年损耗直接干到了30%，实在是受不了不插电时电池就以肉眼可见的速度掉电了。按说Book3的配置在现阶段也算是中高端水平了，但是受限于笔记本平台散热结构的限制，CPU、GPU的性能其实都是大大折扣的。

笔记本显然已经不能满足我现在对电脑的需求了，于是乎决定出了笔记本，组装人生第一台主机。

都是借口，主要缘由还是笔记本已经玩儿厌了，想要折腾折腾台式。 ::(你懂的)

搞一个台式其实是我多年的梦想了，但是受限于环境的因素，一直都在学校，寒暑假都需要回家什么的，而读研之后更是需要经常出差啥的，还是笔记本方便点。于是这个梦想就是一直处于搁浅状态。

直到最近，决定圆了自己的这个梦，实在是受不了笔记本那残血的性能了！！！ ::(怒)

再见了，Microsoft Surface Book 3.

[photos]

[/photos]

准备

其实配一台主机，最重要的就是你的预算和你的需求，你的需求决定了你对它的期待是什么，而你的预算决定了你能购买到的配置能不能达到你对它的期待。一台主机的性能以及你对它的满意度基本上是与它的价格成正比的。

所以当你决定配一台台式机，或者推广到购买任何一样电子产品的时候都是这样的。最重要的就是你的预算与你的需求。

由于我准备出掉手里的Surface Book 3，因此还算有一笔充足的资金来支持；至于我的需求的话，我基本就是日常Office办公+Auto CAD绘图+一些大型3A游戏（地平线、战地5和GTA等）。

Go…

机箱

机箱代表了一个台式机颜值，同时也决定了其配件在选配上的范围。不同大小的机箱对主板、显卡、电源以及风扇或者其他配件都有一定的尺寸要求或者限制。所以第一步就是要根据你的需求挑选一个适合你的机箱。

有什么样的需求就有什么样的市场，现阶段市面上各种大小、尺寸、结构的机箱种类不计其数，大家可以根据自己的想法去淘宝、京东搜索。

最简单粗暴的一句总结机箱：大机箱容易装机（理线），对各种配件的支持度好，方便后续升级，散热也好，但是有可能看起来傻大黑粗的（也并不完全都是，很大看着还好看的一般都很贵）；小机箱方便携带，但是对主板、显卡、电源等配件的大小尺寸可能有一定的约束。

至于有些想买小机箱，但是却担心自己不知道如何选购自己机箱合适的配件的同学完全不用担心，因为你购买机箱的时候，机箱的详情页面都会详细地告诉你本机箱对各种配件的支持度以及限制，你只要按照上面的要求来就行了（如下图，我购买的机箱详情页面都有对各种配件支持度的详细介绍）。

于是乎我的第一步是挑选了一款自己喜欢的机箱。我的要求如下：

  1. 侧板肯定要玻璃透明的，赤裸相见才是最真诚的 ::(滑稽) ；
  2. 机箱大小的话，首先不要傻大黑粗的ATX，在我眼里越小的东西才越精致，而ITX又太小，以后升级选配件的时候会受到限制，于是我的最佳选择就是介于两者之间的MATX；
  3. 机箱材料的话我还是比较喜欢铝合金的。

介于以上三点要求，很容易在某宝就找到了小喆优品的一款型号为C4的机箱。机箱的配置信息如下：

[photos]

[/photos]

在安装过程中发现，这款机箱的显卡固定螺丝，以及背板2.5寸硬盘固定螺丝的开模都很不准；还有个问题就是熏黑的玻璃周围有黑色图层，很不结实，用指甲都可以扣下来。 ::(喷)

主板+CPU

台式机的性能与同平台的笔记本之间的差距存在非常大的差距（得益于台式机良好的散热系统），为了给显卡留充足的预算，主板+CPU我的目标就是一般水平就行。

虽然AMD Yes，但是出于我日常要使用部分较小众的软件可能对AMD平台的支持度并不是很好，而且12代酷睿在AMD的威胁下，一下挤出了往年好几年的牙膏，性能提升也算是可以的。因此在CPU的选择上我还是决定使用英特尔平台。

额外需求的话，我平时电脑有外接的蓝牙设备，以及平时网线连接的校园网不够稳定，有时候需要切换至手机热点。因此对于主板我需要有蓝牙+WIFI的支持。

最后综合价格等因素选择了英特尔【12代】i5 12400F 6核12线程 无核显+华硕TUF B660M-PLUS WIFI D4组合。

需要注意的是，这个CPU是不带核显的，同时这个主板也是没有集显的，这也就意味着你的电脑必须有显卡，而且把显示器的HDMI线插在显卡上才能点亮屏幕。

硬盘

硬盘的话因为平时我对于资料的话都喜欢使用云备份，因此主硬盘还是以系统+应用为多，选择了金士顿 M.2 1T，另外机箱到手后发现背板处有安装2.5寸硬盘的位置，因此又在闲鱼入手了一块希捷 2.5寸 2T的硬盘。

硬盘建议还是上有保障的大厂产品吧，上一年在怀宁干项目的时候就亲眼目睹了虎子的笔记本硬盘突然挂掉，然后导致自己画了一个星期的图就没了。 ::(小乖)

显卡

显卡的话，平时使用AutoCAD也会用一点点，但是最重要的还是玩儿游戏。半个月前，英伟达20系列显卡没有比30系列便宜多少，但是30系列的性能要比20系列提高了很多，于是还是咬咬牙准备上了30系列。最后选择了一款白色主题的显卡——七彩虹（Colorful）iGame RTX3060 AD W OC 12G。

这特么刚买完半个月，就宣布显卡要大降价了！！！ ::(怒)

买回来的时候没在意，到手之后才发现，显卡的灯光处的屏幕上有明显的划痕。明显有问题，但是金手指处还是挺新的，第一时间联系了售后，售后倒是挺负责，同意寄回。具体怎么处理就不清楚了。无良商家七彩虹，官方翻新无疑。 ::(怒)

[/photos]

电源

因为我选择的机箱体积较小，留给理线的空间并不大，因此选择了全模组电源，方便理线。但是电源买回来之后发现所带的电源线也太长了，一怒之下又在淘宝定制了电源线（本人重度强迫症患者）。

当然，最后的理线确实也是非常成功的。人民币果然能解决很多问题！

[photos]

[/photos]

分享下定制线的长度数据：

  1. 主板 24pin 1条 38 cm；
  2. cpu 8pin 1条 68 cm；
  3. 显卡 6+2pin 2条 38 cm；
  4. sata线（一拖一） 1条 35 cm。

注意：本定制线数据只适用于本机箱（小喆优品C4）搭配本电源（鑫谷 GM750W 冰山版）！

内存

皇家戟产品的性价比不高，但是我买东西基本上是颜值党，实在是抵抗不了镜面+RGB的诱惑。本来想上16*2的32G套装，但是最后综合考虑了预算，还是选择了8*2 16G套装。以后有钱了再加吧！ ::(胜利)

风扇

听说RGB风扇可以增加电脑500%的性能，于是乎果断选择了。

[photos]

[/photos]

机箱上下共安装了五把支持RGB神光同步的12 mm风扇，性能也不知道咋样，但是是真的好看！！！

可以看到后板的两把是不带RGB灯光的，我找遍全网也没有找到支持神光同步的白色9 mm风扇。按理说有需求就会有市场啊？！

有了解的朋友可以指点一下。

CPU散热器

受限于机箱限制，塔式散热器支持的最高高度为136 mm，因此经过综合对比，选择了乔思伯 CR-1400，但是买回来才发现这一款是不支持神光同步的，有点小遗憾。但是当时急于装机也就懒得退了，凑合凑合用吧。

装好了点亮机器才发现这个散热器是真的辣鸡，风扇动平衡估计都没做过，摆幅很大。果然便宜没好货，以后还是有时间换个利民的吧。
